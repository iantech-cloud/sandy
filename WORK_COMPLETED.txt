OBJECTIVE:
  Investigate why users are paying for account activation (KES 95) but their
  accounts are not being activated properly. Ensure authentication flow works
  correctly and users can login after activation.

SCOPE:
  • Authentication system (auth.ts)
  • Payment processing (M-Pesa callback)
  • Activation flow (activation.ts)
  • Session management (NextAuth)
  • User login flow

================================================================================
                         FINDINGS SUMMARY
================================================================================

✅ ROOT CAUSE IDENTIFIED:
  JWT token was created at login and NEVER refreshed when database was updated
  after payment. The token remained stale with is_active: false indefinitely.

✅ CRITICAL ISSUE:
  User Session Callback was reading from JWT token instead of database.
  Since JWT token wasn't updated after payment, users remained locked out.

✅ SECONDARY ISSUES FOUND:
  1. Activation status logic used OR instead of AND (too loose)
  2. No timestamp tracking for activation payments
  3. No explicit session refresh after payment
  4. Session callback had no logging for debugging
