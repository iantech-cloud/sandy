SIGN-UP PAGE PHONE NUMBER VALIDATION FIX
==========================================

ISSUE FIXED:
- Sign-up page was only accepting 9-digit phone numbers (e.g., 712345678)
- Users could not enter 10-digit numbers starting with 0 (e.g., 0712345678)
- This rejected the standard Kenyan phone format

CHANGES MADE:

1. FILE: app/auth/sign-up/SignUpContent.tsx
   
   A. ADDED IMPORT:
      - Imported formatPhoneNumber utility for proper phone normalization
   
   B. UPDATED VALIDATION REGEX (lines 130-141):
      OLD: Only accepted /^[0-9]{9}$/ (9 digits)
      NEW: Accepts three formats:
           - /^0[0-9]{9}$/ (10 digits starting with 0: 0712345678) ✓
           - /^[0-9]{9}$/  (9 digits: 712345678) ✓
           - /^254[0-9]{9}$/ (12 digits with country code: 254712345678) ✓
   
   C. UPDATED PHONE FORMATTING (line 158):
      OLD: Blindly prepended +254: `+254${formData.phone.replace(/\s/g, '')}`
           This would create: +2540712345678 (WRONG - duplicate leading 0)
      NEW: Uses formatPhoneNumber() utility which properly handles all formats
           - 0712345678 → +254712345678 ✓
           - 712345678 → +254712345678 ✓
           - 254712345678 → +254712345678 ✓
           - +254712345678 → +254712345678 ✓
   
   D. UPDATED UI (lines 389-404):
      - Changed label from "Phone Number (9 digits)" to "Phone Number"
      - Updated placeholder from "254XXXXXXXXX without country code" to "e.g., 0712345678"
      - Added helper text: "Enter 10 digits starting with 0 (e.g., 0712345678)"
