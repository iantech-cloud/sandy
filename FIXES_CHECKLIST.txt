================================================================================
  THE FIX - WHAT WAS WRONG
================================================================================

PROBLEM:
  1. User pays for activation
  2. Backend updates database: is_active = true ✓
  3. But user's JWT token still says: is_active = false ✗
  4. Session shows "not activated" even though payment succeeded
  5. User cannot access account

ROOT CAUSE:
  NextAuth JWT token was created at login and NEVER refreshed
  Callbacks didn't re-read from database after payment
  No session refresh endpoint existed

SOLUTION:
  1. Create /api/auth/refresh-session endpoint
  2. Frontend calls it after activation payment
  3. Endpoint fetches fresh user data from DB
  4. Session immediately updated with is_active: true
  5. User can access account without logging out again
