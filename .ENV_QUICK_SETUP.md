# M-PESA Daraja .ENV Quick Setup

## Minimum Required for STK Push Payments

Copy and paste this into your `.env.local` or `.env` file:

```env
# ============================================
# DARAJA OAUTH (Required)
# Get from: https://developer.safaricom.co.ke/
# ============================================
DARAJA_CONSUMER_KEY=your-consumer-key-here
DARAJA_CONSUMER_SECRET=your-consumer-secret-here

# ============================================
# DARAJA STK PUSH (Required)
# Get from: M-PESA test app in Daraja portal
# ============================================
DARAJA_BUSINESS_SHORT_CODE=600123
DARAJA_PASSKEY=your-passkey-here

# ============================================
# CALLBACK URL (Required)
# Where M-PESA sends payment notifications
# ============================================
DARAJA_CALLBACK_URL=https://your-domain.com/api/payments/daraja/callback
NEXT_PUBLIC_BASE_URL=https://your-domain.com
```

## How to Get These Values

### Step 1: Create Daraja Account
1. Go to https://developer.safaricom.co.ke/
2. Sign up for a Daraja account
3. Create a new app

### Step 2: Get Consumer Key & Secret
1. In Daraja app, go to "Credentials"
2. Copy **Consumer Key** → paste as `DARAJA_CONSUMER_KEY`
3. Copy **Consumer Secret** → paste as `DARAJA_CONSUMER_SECRET`

### Step 3: Get Passkey & Short Code
1. In Daraja app, create an "M-PESA Test App"
2. Copy **Passkey** → paste as `DARAJA_PASSKEY`
3. Copy **Business Short Code** → paste as `DARAJA_BUSINESS_SHORT_CODE`

### Step 4: Setup Callback URL
1. Your callback URL is where payments results are sent
2. For development: Use ngrok `https://abc123.ngrok.io/api/payments/daraja/callback`
3. For production: Use your domain `https://hustlehubafrica.com/api/payments/daraja/callback`
4. Register this URL in Daraja portal under "API Configuration"

## Optional: Payout Features

Only add if implementing B2C payouts, B2B transfers, etc.:

```env
# For business payouts (optional)
DARAJA_INITIATOR_NAME=api_operator
DARAJA_INITIATOR_PASSWORD=your-encrypted-password

# For encryption (optional)
MPESA_PUBLIC_KEY_SANDBOX=-----BEGIN PUBLIC KEY-----\n...
MPESA_PUBLIC_KEY_PRODUCTION=-----BEGIN PUBLIC KEY-----\n...
```

**Note**: These are optional. STK Push works without them.

## What NOT to Add

Do NOT add these anymore (they're deprecated):

```env
# ❌ DEPRECATED - DO NOT USE
COOP_BANK_BASIC_AUTH=...
COOP_BANK_OPERATOR_CODE=...
MPESA_SHORTCODE=...
MPESA_CONSUMER_KEY=...
```

All payments now use M-PESA Daraja APIs.

## Verify It's Working

After updating .env:

1. Restart your dev server (`npm run dev`)
2. Try a test payment on the app
3. Check your browser console for payment prompt
4. Complete payment on M-PESA
5. Check server logs for callback received

## Common Issues

### "Daraja credentials not configured"
- Check `.env` file has all 4 required variables
- Restart dev server after updating .env
- Verify no typos in variable names

### "Callback URL not accessible"
- For local: Use ngrok and register temporary URL
- For production: Ensure domain has SSL (HTTPS)
- Whitelist Safaricom IPs if using firewall

### "Invalid phone number"
- Use format: 254XXXXXXXXX or 0XXXXXXXXX
- App automatically converts 0XXX to 254XXX
